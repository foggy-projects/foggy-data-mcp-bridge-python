当前对齐结论：

- Java 8.3.0.beta 设计文档已记录 CTE / 派生 / Join / Union 双端实施完成，包含自动 CTE 去重与 compose query plan-aware 编译路线。

- 显式命名 CTE、递归 CTE、跨数据源 live detection 的完整运行时阻断仍不属于当前签收范围。

保留项：

- 如后续需要把 CTE 能力从 accepted baseline 扩到显式 named CTE / recursive CTE，需要新开契约与双端 follow-up。

### TimeWindow

当前对齐结论：

- Java 8.3.0.beta `timeWindow` 已签收，Python v1.5 parity lane 已完成运行时对齐。
- Java 8.4.0.beta 明确 `timeWindow + calculatedFields` 契约，Java 8.5.0.beta 已实现后置 scalar calculatedFields；Python 已按该子集完成实现和验收。
- 当前两个引擎对文档要求内的 rolling / cumulative / comparative / post scalar calculatedFields 子集已对齐。

当前签收的允许能力：

- `timeWindow` 输出列之上的后置 scalar calculatedFields。
- 表达式引用 timeWindow 输出列，例如 `salesAmount__prior`、`salesAmount__diff`、`salesAmount__ratio`、`salesAmount__rolling_7d`。
- 最终层 `ORDER BY` / `LIMIT`，并支持按 post calc `name` 排序、输出使用 `alias`。

当前签收的 fail-closed 能力：

- `targetMetrics` 不允许引用请求级 calculatedFields。
- 后置 calculatedFields 不允许 `agg`。
- 后置 calculatedFields 不允许 `partitionBy` / `windowOrderBy` / `windowFrame`。
- 后置表达式只能引用 timeWindow 最终输出列。
