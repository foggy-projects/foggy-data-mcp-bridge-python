## Gaps

### G1：无真实数据库执行链路测试（minor，非阻断）

- **现象**：所有测试都是单元/契约级断言（对生成的 SQL 字符串做 `in` 或等值判断），**没有对生成的 SQL 实际跑 MySQL/Postgres/SQLite/SQL Server** 的 integration 测试
- **影响**：方言翻译若有细微语法错误（如 `TO_CHAR` 格式串字符在 Postgres 实际不认），本轮测试捕捉不到
- **缓解**：
  - Phase 1 的翻译是**直接镜像 Java 已在生产跑过** 的 `buildFunctionCall` 逻辑，风险低
  - `tests/test_mcp/` + 既有 docker 化 MySQL 测试（见 CLAUDE.md 提到的 13306 端口）会对主流程 SQL 兜底
  - Phase 3 feature flag 默认关闭，opt-in 才启用，生产风险隔离
- **评估**：**允许带风险进入验收**；后续若启用 `use_ast_expression_compiler=True` 作为默认，再补真实 DB 跑通验证

### G2：无 Java ↔ Python 跨语言对照自动化测试（minor，非阻断）

- **现象**：契约对齐是通过 progress 文档的**人工对照表**呈现的，没有一个测试程序同时调 Java 和 Python 对同一输入求输出并比对
- **影响**：Java 侧后续若改 `buildFunctionCall`，Python 这边不会主动触发失败
- **缓解**：
  - 跨仓跨语言测试基础设施缺失（非本轮 scope）
  - 日常 Java 侧变更会触发 Java 自己的 `DialectFunctionTranslationTest`，通过 PR review 同步 Python
  - v1.5 Phase 1 / v1.4 的用例是**按 Java 测试清单手工镜像**的，首次对齐时密度足够
- **评估**：**非阻断**。长期可考虑开 `docs/` 下做一份 cross-language contract test harness

### G3：Phase 3 AST visitor 的 79% 行覆盖（nit，非阻断）

- **现象**：`fsscript_to_sql_visitor.py` 247 行中有 51 行未覆盖
- **分析**：全部未覆盖行都是 **defensive 错误分支**：
  - `"AST node not supported by SQL visitor"` raise（遇到未知节点类型）
  - `"Cannot emit literal of type X"` raise（遇到未知字面量类型）
  - `_emit_literal` 通用字面量 fallback（实际走不到，因为具体字面量已优先分派）
  - `_collect_member_path` 的 non-variable base 错误
- **影响**：这些分支就是"不该被触发"的最后防线；给它们补测试 ROI 极低
- **评估**：**不建议补**。若未来添加新 AST 节点类型，自然会触发这些分支，届时再加

### G4：无 performance / load 测试（nit，非阻断）

- **现象**：Phase 1（每函数 dialect 路由）/ Phase 2（每查询 Kahn sort）/ Phase 3（opt-in AST parse）都引入了新开销
- **分析**：
  - Phase 1：`_get_function_mappings()` 之前每次调用重建 dict，simplify 修复后改为类级常量 → 反而比 pre-v1.5 更快
  - Phase 2：Kahn 算法 O(V+E)，典型 QM 计算字段 < 10 个，纳秒级
  - Phase 3：AST parse O(n)，仅在 opt-in 启用时运行
- **评估**：**不需要补**。新开销都是 bounded 且数量级远低于 SQL 执行本身

### G5：`use_ast_expression_compiler=True` 无 production-like manual evidence（minor，非阻断）

- **现象**：AST flag 开启后用 DSL `calculatedFields` 跑真实查询的 manual evidence 未落盘
- **缓解**：
  - 76 AST 测试已覆盖 SQL 生成端到端（通过 mode=validate 模拟真实 query 路径）
  - Fallback 设计确保即使 AST 失败也能降级
  - Flag 默认 False —— 生产默认路径完全未变
- **评估**：**允许带风险**。若未来翻默认到 True，补一次完整手工跑通记录即可
